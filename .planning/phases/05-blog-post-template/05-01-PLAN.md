# Phase 5: Blog Post Template - Plan 01

## Objective

Enhance the existing blog post template with sports media styling, author bio section, and category/tag display. The current template has the structure but needs Clemson branding and missing components.

## Current State Analysis

**What Already Exists:**
- `frontend/app/blog/[slug]/page.tsx` - Full blog post page with:
  - Featured image display
  - Reading progress bar
  - Social sharing (ShareButton)
  - Related posts section (BlogCard)
  - Reading time estimate
  - Structured data (JSON-LD)
  - Basic author name display

**What Needs to Be Added:**
1. **AuthorBio component** - Full author section with avatar, name, bio text, social links
2. **Category/Tag display** - Show post categories and tags with links
3. **Clemson Sports Media styling** - Orange/purple branding, font-heading typography

## Execution Context

**Files to modify:**
- `frontend/app/blog/[slug]/page.tsx` - Main blog post template
- `frontend/lib/wordpress.ts` - May need to fetch additional author/category data

**Files to create:**
- `frontend/components/AuthorBio.tsx` - New component
- `frontend/components/PostMeta.tsx` - Category and tag display component

**Reference files:**
- `frontend/components/AuthorByline.tsx` - Existing small byline (for patterns)
- `frontend/components/CategoryBadge.tsx` - Existing badge styling
- `frontend/app/page.tsx` - Homepage styling patterns

## Tasks

### Task 1: Create AuthorBio Component
Create a new AuthorBio component for the bottom of blog posts.

**File:** `frontend/components/AuthorBio.tsx`

**Requirements:**
- Large avatar (64px or 80px)
- Author name in font-heading, bold
- Bio text (if available from WordPress)
- Social links (Twitter, LinkedIn) - optional
- Clemson purple background option or border accent
- Responsive layout (stack on mobile)

**Props interface:**
```tsx
interface AuthorBioProps {
  name: string;
  avatar?: string | null;
  bio?: string;
  twitterUrl?: string;
  linkedinUrl?: string;
  className?: string;
}
```

### Task 2: Create PostMeta Component
Create a component to display categories and tags.

**File:** `frontend/components/PostMeta.tsx`

**Requirements:**
- Display categories as orange badges (using existing CategoryBadge style)
- Display tags as smaller pills or links
- Link to category/tag archive pages
- Horizontal layout with wrapping
- Separator between categories and tags

**Props interface:**
```tsx
interface PostMetaProps {
  categories: Array<{ id: number; name: string; slug: string }>;
  tags: Array<{ id: number; name: string; slug: string }>;
  className?: string;
}
```

### Task 3: Update WordPress Library for Category/Tag Data
Ensure the `getPost` function returns full category and tag objects.

**File:** `frontend/lib/wordpress.ts`

**Requirements:**
- Use `_embed` parameter to get category/tag details
- Extract category names, slugs, IDs from embedded data
- Extract tag names, slugs, IDs from embedded data
- Update WPPost type if needed

### Task 4: Apply Clemson Sports Media Styling to Blog Post
Update the blog post page with brand styling.

**File:** `frontend/app/blog/[slug]/page.tsx`

**Changes:**
1. **Header section:**
   - Use `font-heading` for title
   - Add orange accent line under title
   - Style meta info with Clemson colors

2. **Content area:**
   - Style prose headings with `font-heading`
   - Orange link colors
   - Blockquote styling with purple left border

3. **Add AuthorBio section:**
   - Place after article content, before related posts
   - Use full-width container with background

4. **Add PostMeta section:**
   - Place below article content, above AuthorBio
   - Show categories and tags

5. **Related Posts section:**
   - Add orange accent header line
   - Use `font-heading` for section title
