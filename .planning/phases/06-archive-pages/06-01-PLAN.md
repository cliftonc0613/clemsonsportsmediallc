# Phase 6: Archive Pages - Plan 01

## Objective

Create category and tag archive pages with pagination for browsing posts by taxonomy. These pages will be linked from the blog post template's PostMeta component.

## Current State Analysis

**What Already Exists:**
- `frontend/app/blog/page.tsx` - Blog listing with filters (no pagination)
- `frontend/lib/wordpress.ts` - `getPostsByCategorySlug()`, `getCategoryBySlug()`, `getTagBySlug()`
- `frontend/components/BlogCard.tsx` - Post card component
- PostMeta links to `/category/[slug]` and `/tag/[slug]`

**What Needs to Be Created:**
1. Category archive page (`/category/[slug]`)
2. Tag archive page (`/tag/[slug]`)
3. Pagination component
4. Archive header component (optional - for consistent styling)

## Execution Context

**Files to create:**
- `frontend/app/category/[slug]/page.tsx` - Category archive
- `frontend/app/tag/[slug]/page.tsx` - Tag archive
- `frontend/components/Pagination.tsx` - Reusable pagination

**Files to modify:**
- `frontend/lib/wordpress.ts` - Add pagination metadata support

## Tasks

### Task 1: Create Pagination Component

**File:** `frontend/components/Pagination.tsx`

**Requirements:**
- Page numbers with current page highlighted
- Previous/Next buttons
- Clemson styling (orange active state)
- Responsive (collapse to prev/next on mobile)
- URL-based navigation (searchParams)

**Props interface:**
```tsx
interface PaginationProps {
  currentPage: number;
  totalPages: number;
  basePath: string;
  className?: string;
}
```

### Task 2: Update WordPress Library for Pagination

**File:** `frontend/lib/wordpress.ts`

**Requirements:**
- Create function to get total pages from API response headers
- Add `getPostsWithPagination()` that returns both posts and pagination info

### Task 3: Create Category Archive Page

**File:** `frontend/app/category/[slug]/page.tsx`

**Requirements:**
- Dynamic route for category slug
- Fetch category info and posts
- Display category name as page title
- Orange accent header styling (like sport sections)
- Grid of BlogCard components
- Pagination at bottom
- SEO metadata (title, description, Open Graph)
- Breadcrumbs: Home > Category > [Category Name]

**Layout:**
```
┌─────────────────────────────────────┐
│ [Category Name] Header              │
│ with watermark background           │
├─────────────────────────────────────┤
│ Posts Grid (3 columns)              │
│ ┌─────┐ ┌─────┐ ┌─────┐            │
│ │Card │ │Card │ │Card │            │
│ └─────┘ └─────┘ └─────┘            │
│ ...                                 │
├─────────────────────────────────────┤
│ Pagination: < 1 2 3 ... 10 >       │
└─────────────────────────────────────┘
```

### Task 4: Create Tag Archive Page

**File:** `frontend/app/tag/[slug]/page.tsx`

**Requirements:**
- Similar structure to category archive
- Tag-specific styling (smaller header, no watermark)
- Display tag name and post count
- Grid of BlogCard components
- Pagination
- SEO metadata

### Task 5: Test and Verify

**Verification steps:**
1. Navigate to `/category/football` (or any category)
2. Verify category name displays correctly
3. Verify posts are filtered to that category
4. Test pagination (if enough posts)
5. Navigate to `/tag/[tag-slug]`
6. Verify tag archive works similarly
7. Test links from blog post PostMeta component

## Success Criteria

- [ ] Category archive page displays posts filtered by category
- [ ] Tag archive page displays posts filtered by tag
- [ ] Pagination works correctly
- [ ] Pages use Clemson Sports Media styling
- [ ] SEO metadata generated for archives
- [ ] Links from PostMeta work correctly

## Output

After completion:
1. `/category/[slug]` routes work for all categories
2. `/tag/[slug]` routes work for all tags
3. Pagination enables browsing large post collections
4. Consistent styling with rest of site
